Store model credentials in .env files

Published

March 9, 2026

On this page

  • Why is this recommended?
  • Prerequisites
  • 1. Store credentials in an .env file
  • 2. Use credentials in your Jupyter Notebook
  • What's next

Learn how to store model identifier credentials in a .env file instead of using inline credentials, allowing you to follow best practices for security when running Jupyter Notebooks.

Why is this recommended?

Storing credentials in a .env file is considered a best practice for security. Embedding credentials directly within the code makes them more susceptible to accidental exposure when sharing code or collaborating on models.

Keeing model credentials in a separate file also allows for precise access control and ensures that sensitive credentials are not publically accessible.

Prerequisites

1. Store credentials in an .env file

  • a. Get your code snippet
  • b. Create an .env file

In the ValidMind Platform, retrieve the code snippet for your model:

  1. In the left sidebar, click Inventory.

  2. Select a model by clicking on it or find your model by applying a filter or searching for it.3

  3. In the left sidebar that appears for your model, click Getting Started.

  4. (Optional) Select the document you want to automatically upload test results to.4

  5. Locate the code snippet and click Copy snippet to clipboard.

  1. Create a new file in the same folder as your notebook and name it .env.

    This is a hidden file, so you may need to change your settings to view it.

  2. Use the code snippet from your clipboard to build the credentials in your .env file in the following format:5

    VM_API_HOST=<api_host>
VM_API_KEY=<api_key>
VM_API_SECRET=<api_secret>
VM_API_MODEL=<model>

To define the document to connect the ValidMind Library to, or to enable monitoring, include these in your vm.init() snippet within your Jupyter Notebook.6

2. Use credentials in your Jupyter Notebook

  1. After installing the ValidMind Library within your Jupyter Notebook, insert this code snippet in a code cell above your model identifier credentials:

    %load_ext dotenv
%dotenv dev.env

  2. Remove the inline credentials from vm.init().

  • with
  • with 2

The updated code cell should now look like this:

%load_ext dotenv
%dotenv .env

import validmind as vm

vm.init(
   document="document-key",
)

To enable monitoring when you are storing credentials in an .env file:7

%load_ext dotenv
%dotenv .env

import validmind as vm

vm.init(
   document="document-key",
   monitoring=True
)

  1. Run the cell.

    Instead of using inline credentials, this code cell will now load your model credentials from a .env file.
