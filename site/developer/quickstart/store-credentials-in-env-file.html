Store record credentials in .env files

Published

June 8, 2026

Learn how to store record (model) identifier credentials in an .env file instead of using inline credentials, allowing you to follow best practices for security when running Jupyter Notebooks.


NoneWhy is it recommended to store credentials in an .env file?

Storing credentials in a .env file is considered a best practice for security. Embedding credentials directly within the code makes them more susceptible to accidental exposure when sharing code or collaborating on records.

Keeping record credentials in a separate file also allows for precise access control and ensures that sensitive credentials are not publicly accessible.

Prerequisites

1 Register records in the inventory

2 Manage inventory record stakeholders

Before you follow the steps on this page, we recommend that you first complete our introductory guide on how to Install and initialize the ValidMind Library.

1. Store credentials in an .env file

  • a. Get your code snippet
  • b. Create an .env file

In the ValidMind Platform, retrieve the code snippet for your record:

  1. In the left sidebar, click Inventory.

  2. Under the record type drop-down, select the type of record you want to retrieve the code snippet for.3

  3. Select a record by clicking on it or find your record by applying a filter or searching for it.4

  4. In the left sidebar that appears for your record, click Getting Started.

  5. Select the document you want to automatically upload test results to.5

  6. Click Copy snippet to clipboard.

3 Manage inventory record types

4 Working with the inventory

5 Managing documents

  1. Create a new file in the same folder as your notebook and name it .env.

    This is a hidden file, so you may need to change your settings to view it.

  2. Use the code snippet from your clipboard to build the credentials in your .env file in the following format:6

    VM_API_HOST=<api_host>
VM_API_KEY=<api_key>
VM_API_SECRET=<api_secret>
VM_API_MODEL=<model>

6 For example, if your credentials look like this:

import validmind as vm

vm.init(
api_host = "https://api.prod.validmind.ai/api/v1/tracking",
api_key = "key123",
api_secret = "secret456",
document = "document-key",
model = "model789"
)

Then your .env file should look like this:

VM_API_HOST=https://api.prod.validmind.ai/api/v1/tracking
VM_API_KEY=key123
VM_API_SECRET=secret456
VM_API_MODEL=model789

To define the document to connect the ValidMind Library to, or to enable monitoring, include these in your vm.init() snippet within your Jupyter Notebook.7

7 Use credentials in your Jupyter Notebook

2. Use credentials in your Jupyter Notebook

  1. After installing the ValidMind Library within your Jupyter Notebook,8 insert this code snippet in a code cell above your record (model) identifier credentials:

    %load_ext dotenv
%dotenv dev.env

  2. Remove the inline credentials from vm.init() after importing the library:

    • Without ongoing monitoring
    • With ongoing monitoring

    Example updated code cell:

    %load_ext dotenv
%dotenv .env

import validmind as vm

vm.init(
   document="document-key",
)

    Example updated code cell including ongoing monitoring:9

    %load_ext dotenv
%dotenv .env

import validmind as vm

vm.init(
   document="monitoring",
   monitoring=True
)

  3. Run the cell.

    Instead of using inline credentials, this code cell will now load your record credentials from a .env file.
