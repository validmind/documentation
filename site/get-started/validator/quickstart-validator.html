Quickstart — Validation

20 minutes

Published

June 1, 2026

Use the ValidMind Library to run tests and validate a record (model), then attach your test results as evidence to your validation report in the ValidMind Platform.

Before you begin

1 Register with ​ValidMind

2  Log in to ​ValidMind

Register a model for validation

Typically, you will be asked to validate a champion that has already been independently registered in the inventory and submitted to you. In this guide, we’ll use a model-type record for our example:

1. Register a sample model

For this quickstart, you’ll register a dummy model first and assign yourself as the validator to familiarize you with the ValidMind Platform interface:

  1. In the left sidebar, click Inventory.

  2. Under the record type drop-down, select Model.3

  3. Click Register Model.

  4. Enter the model name and select any option for the following required fields:

    • business unit — For example: Finance
    • prelimnary risk tier — For example: 1

  5. Click Next to continue to assignment of inventory record stakeholders.

  6. Select your own name under the record owner drop-down — don’t worry, we’ll adjust these permissions next for validation.

  7. Click Register Model to add the model to your inventory.

3 Manage inventory record types

2. Assign validator credentials

In order to log tests as a validator instead of as a developer, we’ll need to adjust your inventory record stakeholder permissions.

On the model details page after you’ve registered your model:

  • Remove yourself as owner
  • Remove yourself as developer
  • Add yourself as validator
  1. Click on the owners tile, then click on the x next to your name.
  2. Click Save to apply your changes to that role.
  1. Click on the developers tile, then click on the x next to your name.
  2. Click Save to apply your changes to that role.
  1. Click on the validators tile, and select your name from the drop-down menu.
  2. Click Save to apply your changes to that role.

3. Select a report template

  1. In the left sidebar that appears for your model, click Documents and select Validation.

  2. Under create document, choose From ​ValidMind Template and then select Generic Validation Report from the template drop-down menu.

  3. Click Use Template to apply the template.

If you cannot locate your Validation document, make sure Validation type documents are enabled for model records and create a new document.4

4 Manage documents

4. Get your code snippet

Before we move on, let’s also retrieve the unique code snippet for your model that enables you to connect your testing environment with the correct document in the ValidMind Platform:

  1. In the left sidebar that appears for your model, click Getting Started.

  2. Select Validation from the document drop-down menu.

  3. Click Copy snippet to clipboard.

Using the ValidMind Library

Next, let’s set up the ValidMind Library in your validation environment of choice so we can use it to run and log tests, which helps automate the validation of your records.

Our companion notebook, Quickstart for validation,5 walks you through the steps of installing the library in your environment, initializing the library for use, and running data quality and model evaluation tests that log results to the ValidMind Platform.

5 Quickstart for validation

Access the quickstart notebook

You can either load the hosted version of our notebook, or download the notebook to run in your own local environment:

Open the hosted notebook (recommended)

  1. Sign in to JupyterHub with your ​ValidMind login.

  2. Open Quickstart for validation and run the quickstart notebook.

Download the notebooks

  • Option 1: Clone the GitHub repository
  • Option 2: Download the code samples

  1. Open a terminal or command prompt.

  2. Navigate to the directory where you want to clone the repository.

  3. Run the following command:

    git clone https://github.com/validmind/validmind-library.git

  4. After the cloning process is complete, open notebooks/quickstart/quickstart_validation.ipynb in your validation environment and run the quickstart notebook..

  1. Download our selection of code samples: notebooks.zip

  2. Navigate to the location where you saved notebooks.zip and extract the contents.6

  3. Navigate to the folder where you extracted the files.

  4. Open notebooks/quickstart/quickstart_validation.ipynb in your validation environment and run the quickstart notebook.

6 Depending on your operating system, you can right-click on notebooks.zip and select Extract All … or Extract Here, for example.

Work with validation reports

After you successfully run the notebook, return to the ValidMind Platform:

  1. In the left sidebar, click Inventory.

  2. Locate or search for the model you registered for this quickstart and click on it to select it.10

  3. In the left sidebar that appears for the model, click Validation under Documents:

    • Your validation report is broken down into sections, defined by your report templates.11
    • The report includes risk assessment summaries at the overall and per-section level, overviews of your compliance assessments provided within your validation report.12
    • Each sub-section (or guideline) allows you to attach test results as evidence,13 as well as log artifacts based on your evaluation of datasets or records.14

Add evidence to your report

Expand any section of the validation report to attach your test results and drafts of test result summaries generated by the ValidMind Library as evidence by clicking Evidence on any guideline to expand the evidence panel.

For example:

  1. Within your validation report, click on Class Imbalance Assessment.

  2. Under the Class Imbalance Assessment guideline click Link Evidence, then select Validator Evidence as the evidence type.

  3. Select the Class Imbalance test results we logged: ValidMind Data Validation Class Imbalance

  4. Click Update Linked Evidence to add the test results to the validation report.
